## 1. 多模态参考的 4 种官方组合

| 组合 | content 列表 |
|------|-------------|
| **图片 + 音频** | `[text, image_url(参考), audio_url(参考)]` |
| 图片 + 视频 | `[text, image_url(参考), video_url(参考)]` |
| 视频 + 音频 | `[text, video_url(参考), audio_url(参考)]` |
| **图片 + 视频 + 音频**（官方示例）| `[text, image_url, video_url, audio_url]` |

> **绘本场景默认用 图片+音频**（v12 范式）。图片+视频+音频 是工业级复杂场景（果茶广告）。
> 官方禁止"文本+音频"和"纯音频"。

## 2. 官方音频要求

| 限制 | 值 |
|------|---|
| 单音频时长 | **[2, 15] 秒** |
| 最多音频数 | **3 段** |
| 所有音频总时长 | **≤ 15 秒** |
| 格式 | wav / mp3 |
| 单文件大小 | ≤ 15 MB |
| 请求体大小 | ≤ 64 MB |
| 传入方式 | URL / Base64 / 素材 ID |

> **绘本 32s 音频必须切分到 ≤15s 段**。`seedance.py --audio` 支持多次传入（每段一个），但单次 API call 总时长 ≤15s。
> 如果绘本超过 3 段（如 5 段），需要**合并相邻段**（每合并段 ≤15s）。

## 3. 音频参考的两种语义

| 语义 | prompt 写法 | 用途 |
|------|------------|------|
| **音色参考**（TTS 替代）| "使用@音频1[具体音色描述]说{台词}" | 角色配音 |
| **BGM 背景音乐**（绘本场景）| "全程使用音频1作为背景音乐。" | 绘本/动画 BGM |

**官方示例**（果茶广告）："全程使用音频1作为背景音乐。第一人称视角果茶宣传广告..."

## 4. 官方内容结构模板（audio_url 段）

```json
{
  "type": "audio_url",
  "audio_url": {
    "url": "https://ark-project.tos-cn-beijing.volces.com/doc_audio/r2v_tea_audio1.mp3"
  },
  "role": "reference_audio"
}
```

**关键**：
- `role` 必须是 `"reference_audio"`
- 缺 `role` 字段 API 返回 `InvalidParameter: reference media mode requires audio role to be reference_audio`
- `type` 必须是 `"audio_url"`（不是 `audio`）

## 5. 官方素材数量上限

| 模态 | 数量上限 |
|------|---------|
| 图片 | 0-9 张 |
| 视频 | 0-3 个 |
| 音频 | 0-3 个 |
| 文本 | 1 段 |

**绘本项目常用 2 张图 + 1 段音频 + 1 段文本** = 完全在限制内。

## 6. 官方"提示词公式"（文档 2 §进阶公式）

```
提示词进阶公式：精准主体 + 动作细节 + 场景环境 + 光影色调 + 镜头运镜 + 视觉风格 + 画质 + 约束条件
```

**绘本项目适配**（v10/v11 实战）：
- 精准主体：`a small round teddy bear`（绘本角色）
- 动作细节：`waves gently up and down with a warm smile`（v11-β 升级）
- 场景环境：`a bright morning meadow`（绘本场景）
- 光影色调：`warm sunrise color palette of cream, brown, orange`（绘本风格）
- 镜头运镜：`the camera slowly drifts forward from a wide shot to a medium shot`（v11-β 升级）
- 视觉风格：`paper-cut collage with watercolor wash, round Q-version teddy bear character`
- 画质：隐含在风格中
- 约束条件：`No human voice, no narration, no subtitles, no text overlays`

## 7. 官方"动作描述要求"（文档 2 §3）

> **绘本项目 v10 → v11-β 升级正好命中这 4 条**：

| 要求 | v10 写法 | v11-β 写法（采纳）|
|------|---------|------------------|
| 肢体细化 + 程度量化 | `holds the X pose` | `waves gently up and down`（具体幅度）|
| 优先低缓连续小动作 | ❌ hold pose（静态） | ✅ 挥手/微笑/呼吸（动态）|
| 补充动作过渡衔接 | 缺 | "with subtle natural breathing motion"（过渡衔接）|
| 情绪具象外化表达 | 缺 | "eyes bright and joyful"（具体情绪）|

**结论**：v11-β 提示词结构（自然动作+镜头漂移）正好是官方推荐写法的"绘本场景实例"。

## 9. 官方"组合参考"页面引用

文档 1 §"基础使用 > 多模态参考" 有一个完整的官方示例（果茶广告）：

```python
content=[
    {"type": "text", "text": "全程使用视频1的第一视角构图，全程使用音频1作为背景音乐。第一人称视角果茶宣传广告，..."},
    {"type": "image_url", "image_url": {"url": "asset://图片1"}, "role": "reference_image"},
    {"type": "video_url", "video_url": {"url": "asset://视频1"}, "role": "reference_video"},
    {"type": "audio_url", "audio_url": {"url": "asset://音频1"}, "role": "reference_audio"}
],
generate_audio=True, ratio="16:9", duration=11, watermark=True
```

> **asset:// 是 OpenClaw/Ark 平台的素材引用简写**，绘本项目用公网 URL 替代。

## 10. 官方"音频结尾噪音"问题（文档 2 §常见问题）

> **典型现象**：视频中包含旁白时，视频片尾容易出现突兀咔哒声、截断杂音。
> **解决方案**：重新生成视频，或使用剪映等剪辑工具，对片尾音轨通过**音量包络线**做音频淡出处理。

**绘本项目适用**：绘本音频的 4 段切分时，最后 1 段（clip4 9s）建议在 ffmpeg 切完后**再做 0.5s 淡出**：

```bash
ffmpeg -y -i audio_clip4.mp3 -af "afade=t=out:st=8.5:d=0.5" audio_clip4_fadeout.mp3
```

**避免**：Seedance 生成时片尾咔哒声 + 4 段拼片时的接缝咔哒声。

### 13.2 根因（5 个，官方文档依据）

| # | 根因 | 官方原话 | 来源 |

## 14. 跨 clip 共享 BGM 主题的"音色 + 配乐"分离（绘本项目必读）

### 14.1 用户的两个核心反馈（2026-06-04）

| 反馈 | 实测现象 | 根因 |
|------|---------|------|
| "BGM 配乐与上传的音频相符" ✅ | v12 生成的 BGM 是钢琴+pizzicato+暖晨光调性 | Seedance 拿上传音频的"风格/音色"作参考生成新 BGM |
| "音频包含了人声，但生成的视频里没有人声" ❌ | v12 生成视频无 TTS 人声 | 音频参考默认是"音色参考"语义，**不是原样播放** |

### 14.2 Seedance 音频处理的两种模式

| 模式 | 用户上传的 mp3 是什么 | Seedance 输出什么 | 用途 |
|------|----------------------|------------------|------|
| **参考模式**（默认）| 风格/音色/旋律参考 | Seedance 自行生成的 BGM，**含风格但无人声原声** | 绘本/动画 BGM 控制 |
| **原样播放**（不推荐 Seedance 实现）| 完整 mp3 轨 | 完整 mp3 轨 | TTS 朗读、保留人声 |

### 14.3 让人声真的播放 → 后期 ffmpeg 替换音频轨

如果用户要"原样播放我的 mp3"（含 TTS 人声）：

**步骤**：
1. **生成视频时不传 `--audio`**：让 Seedance 自带生成 BGM + 音效
2. **生成完成后用 ffmpeg 替换音频轨**：

```bash
ffmpeg -y -i generated_video.mp4 -i original_audio.mp3 \
  -c:v copy -c:a aac -map 0:v:0 -map 1:a:0 \
  final_video_with_user_audio.mp4
```

**绘本项目应用**：用户剪映里合成的 TTS + BGM mp3 → 后期替换 → 绘本视频保留原人声朗读。

### 14.4 v12 / v13 范式选择决策

| 场景 | 推荐范式 | 原因 |
|------|---------|------|
| 绘本 BGM（不要人声）| v12 audio-driven（`--audio` + `--generate-audio true`）| Seedance 用上传音频的风格生成 BGM |
| 绘本 BGM + 人声朗读 | v13 prompt + 后期 ffmpeg 替换音频 | Seedance 生成无 TTS 的 BGM，后期替换原 mp3 |
| 角色配音（用上传音色）| v12 audio-driven + prompt "使用@音频1音色说{台词}" | Seedance TTS 用上传音色生成台词 |
